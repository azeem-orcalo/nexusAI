# NexusAI Project Guide

## Product Context

- Product: NexusAI — AI model marketplace and agent-building platform
- Reference requirements: `requirements.md`
- Design reference: `NexusAI-Dashboard-Updated-15.html` (authoritative)
- Live reference: https://nexusai-db.netlify.app/

## Core Feature Modules

1. **Authentication** — Login/Signup modal, Google OAuth, GitHub OAuth, session management
2. **Guided Onboarding** — 4-step flow (Goal → Audience → Skill → Budget), personalized prompt generation, redirect to Chat Hub
3. **Hero Landing Page** — Animated hero, stats (525+ models, 82K builders, 28 labs, 4.8★), multi-modal search, 14 quick action buttons
4. **Chat Hub** — Model sidebar, chat bubbles, typing indicator, prompt card, inspiration chips, model intro/variation cards, objective wizard, category prompt panel, right sidebar with active model + usage + quick actions
5. **Voice Input** — Speech Recognition API, real-time transcription, silence detection, wave animation, available in hero/chat/marketplace
6. **AI Model Marketplace** — Search, filter pills, AI Labs bar, sidebar facets (provider, pricing, price slider, rating, license), model card grid, empty state, comparison table; 525+ models
7. **Model Detail Modal** — 6 tabs: Overview, How to Use, Pricing, Prompt Guide, Agent Creation, Reviews
8. **Agent Builder** — ACP panel, Agent Library, 5-step wizard (Basics → Tools → Config → Test → Deploy), testing with pass/fail scenarios, deployment (API, embed, Slack, WhatsApp)
9. **Agent Chat View** — Per-agent chat, profile card, short/long-term memory display, tools list, metrics
10. **Task Management** — Create/edit/complete/duplicate/delete tasks, per-task agent conversations, persistent state
11. **Use Case App Discovery** — 60+ apps across 7 categories (Use Cases, Build a Business, Learn, Monitor, Research, Create, Analyze), app detail overlay, launch-to-chat routing
12. **Research Feed** — Two-column layout, research cards, category filter, trending/new model highlights
13. **Dashboard / Analytics** — KPIs (requests, latency, cost), sparkline chart, token usage, satisfaction rating, post-deploy agent metrics
14. **Localization** — 15 languages via nav dropdown
15. **Email Capture** — Weekly digest subscription CTA

## Tech Stack

- Backend: NestJS
- Frontend: React
- Styling: Tailwind CSS
- Database: MongoDB
- Language: TypeScript across frontend and backend

## Design System

- **Fonts**: Syne (headings), Instrument Sans (body)
- **Accent**: `#C8622A` (orange)
- **Backgrounds**: `#F4F2EE` (primary), `#ECEAE4` (secondary), `#E4E1D8` (tertiary)
- **Text**: `#1C1A16` (primary), `#5A5750` (secondary), `#9E9B93` (tertiary)
- **Colors**: Blue `#1E4DA8`, Teal `#0A5E49`, Amber `#8A5A00`, Rose `#9B2042`, Green `#2E9E5B`
- **Border radius**: 8px (sm), 12px (default), 20px (lg), 28px (xl)
- **Shadows**: shadow, shadow-md, shadow-lg
- **Status badges**: NEW (teal), HOT (orange), OPEN SOURCE (blue), BETA (amber)
- **Responsive**: Mobile-first; sidebars collapse at 768px

## Coding Rules

- Follow the DRY principle
- Use functional components for all React UI
- Define TypeScript types for everything
- Prefer clear, composable modules over large files
- Keep business logic separated from presentation logic
- Reuse shared UI and domain types where possible
- Validate inputs at API boundaries
- Keep naming consistent across frontend, backend, and database models

## Scope Notes

- Auth (Google + GitHub OAuth) is fully in scope — implement end-to-end
- Voice input is Must Have — Speech Recognition API required
- Task management is part of Agents view with per-task agent conversations
- Use case app grid (60+ apps, 7 categories) with detail overlays and launch-to-chat routing
- Agent testing and deployment (API, Slack, WhatsApp, embed widget) fully in scope
- Agent memory tracking and tool display required in Agent Chat View
- Admin/back-office, provider model submissions, org/billing/RBAC, saved favorites — NOT in scope

## Agent Roles

### FE Agent

- Owns React and Tailwind implementation
- Builds pages, components, layouts, forms, and client-side state flows
- Translates requirements into responsive, accessible UI
- Uses functional components and typed props throughout
- Coordinates with Integration Agent on API contracts
- Reference file: `.agents/frontend-agent.md`

### BE Agent

- Owns NestJS services, controllers, modules, and MongoDB schema design
- Designs and implements APIs, validation, and business logic
- Maintains clean service boundaries and reusable domain logic
- Ensures all DTOs, entities, and responses are fully typed
- Coordinates with Integration Agent on payload structure and error handling
- Reference file: `.agents/backend-agent.md`

### Integration and QA-Tester Agent

- Owns end-to-end flow validation across frontend and backend
- Verifies API contract alignment between React and NestJS
- Tests critical journeys: auth, browsing, filtering, agent creation, dashboard metrics
- Identifies regressions, edge cases, and integration gaps early
- Confirms delivered work matches `requirements.md`
- Reference files: `.agents/integration-agent.md`, `.agents/qa-tester-agent.md`
